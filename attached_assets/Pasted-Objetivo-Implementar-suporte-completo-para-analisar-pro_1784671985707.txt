Objetivo

Implementar suporte completo para analisar projetos diretamente de repositórios GitHub, sem necessidade de upload de arquivos ZIP.

A implementação deve ser modular, escalável e preparada para suportar futuramente GitLab, Bitbucket e Azure DevOps.

Não alterar a lógica existente do parser ou do pipeline de análise. Apenas criar uma nova fonte de entrada de arquivos.

ETAPA 1 – Auditoria

Antes de escrever qualquer código:

localizar onde atualmente ocorre o upload do ZIP
localizar onde os arquivos são convertidos para ProjectFile[]
localizar onde runAnalysisPipeline() recebe os arquivos
identificar todos os pontos de integração

Não modificar nada antes dessa análise.

ETAPA 2 – Criar arquitetura Repository Provider

Criar a seguinte estrutura:

src/services/repositories/

GitHubProvider.ts

Criar a interface:

export interface RepositoryProvider {

listBranches()

Toda nova origem de projeto deverá implementar essa interface.

Critérios de aceite

A implementação estará concluída quando:

O usuário puder conectar sua conta GitHub.
O sistema listar os repositórios disponíveis.
O usuário puder escolher um repositório e uma branch.
O código-fonte for obtido pela API do GitHub e convertido para ProjectFile[].
O pipeline de análise existente funcionar sem qualquer alteração.
A interface oferecer uma experiência equivalente ao upload de ZIP.
A arquitetura estiver preparada para adicionar novos provedores (GitLab, Bitbucket e Azure DevOps) sem modificar o pipeline.
