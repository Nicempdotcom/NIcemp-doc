## 1) Registrar o usuário logado em cada envio de ZIP

Arquivos envolvidos:
- src/services/storage/types.ts (interface HistoryEntry)
- src/services/storage/mapper.ts (buildAnalysisHistoryEntry e mapProjectMapToEntities)
- src/app/components/upload/UploadProject.tsx
- src/app/providers/AuthProvider.tsx (já existe, expõe useAuth() com `session`)

Passos:
1. Em `types.ts`, adicionar ao `HistoryEntry` um campo opcional:
   uploadedBy?: { id: string; email: string } | null;

2. Em `mapper.ts`:
   - Alterar `buildAnalysisHistoryEntry(projectId, projectName, counts, uploadedBy?)` para aceitar um quarto parâmetro opcional `uploadedBy: HistoryEntry['uploadedBy']` e incluí-lo no objeto retornado.
   - Alterar `mapProjectMapToEntities(projectMap, uploadedBy?)` para receber esse mesmo parâmetro opcional e repassá-lo para `buildAnalysisHistoryEntry`.

3. Em `UploadProject.tsx`:
   - Importar `useAuth` de `@/app/providers/AuthProvider` e chamar `const { session } = useAuth();` no topo do componente.
   - No useEffect que salva no banco (onde chama `mapProjectMapToEntities(projectMap)`), passar o segundo argumento:
     mapProjectMapToEntities(projectMap, session ? { id: session.user.id, email: session.user.email ?? '' } : null)
   - Isso deve entrar como dependência do useEffect (adicionar `session?.user?.id` ao array de dependências) para não quebrar o guard `didSave`.

4. No card de sucesso do upload (`DbSaveStatus` dentro de UploadProject.tsx), exibir, se `session` existir, uma linha extra: "Enviado por {session.user.email}".
